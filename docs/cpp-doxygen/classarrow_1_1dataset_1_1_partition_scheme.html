Detailed Description

Interface for parsing partition expressions from string partition identifiers.

For example, the identifier "foo=5" might be parsed to an equality expression between the "foo" field and the value 5.

Some partition schemes may store the field names in a metadata store instead of in file paths, for example dataset_root/2009/11/... could be used when the partition fields are "year" and "month"

Paths are consumed from left to right. Paths must be relative to the root of a partition; path prefixes must be removed before passing the path to a scheme for parsing.
