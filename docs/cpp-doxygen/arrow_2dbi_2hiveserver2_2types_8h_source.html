27 // Represents a column's type.
28 //
29 // For now only PrimitiveType is implemented, as thase are the only types Impala will
30 // currently return. In the future, nested types will be represented as other subclasses
31 // of ColumnType containing ptrs to other ColumnTypes - for example, an ArrayType subclass
32 // would contain a single ptr to another ColumnType representing the type of objects
33 // stored in the array.
