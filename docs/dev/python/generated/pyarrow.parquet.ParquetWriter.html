  • version ({"1.0", "2.4", "2.6"}, default "1.0") ‚Äì Determine which Parquet logical types are available for use, whether the reduced set from the Parquet 1.x.x format or the expanded logical types added in later format versions. Files written with version=‚Äô2.4‚Äô or ‚Äò2.6‚Äô may not be readable in all Parquet implementations, so version=‚Äô1.0‚Äô is likely the choice that maximizes file compatibility. UINT32 and some logical types are only available with version ‚Äò2.4‚Äô. Nanosecond timestamps are only available with version ‚Äò2.6‚Äô. Other features such as compression algorithms or the new serialized data page format must be enabled separately (see ‚Äòcompression‚Äô and ‚Äòdata_page_version‚Äô).

  • use_dictionary (bool or list) ‚Äì Specify if we should use dictionary encoding in general or only for some columns.

  • use_deprecated_int96_timestamps (bool, default None) ‚Äì Write timestamps to INT96 Parquet format. Defaults to False unless enabled by flavor argument. This take priority over the coerce_timestamps option.

  • coerce_timestamps (str, default None) ‚Äì Cast timestamps to a particular resolution. If omitted, defaults are chosen depending on version. By default, for version='1.0' (the default) and version='2.4', nanoseconds are cast to microseconds (‚Äòus‚Äô), while for other version values, they are written natively without loss of resolution. Seconds are always cast to milliseconds (‚Äòms‚Äô) by default, as Parquet does not have any temporal type with seconds resolution. If the casting results in loss of data, it will raise an exception unless allow_truncated_timestamps=True is given. Valid values: {None, ‚Äòms‚Äô, ‚Äòus‚Äô}

  • data_page_size (int, default None) ‚Äì Set a target threshold for the approximate encoded size of data pages within a column chunk (in bytes). If None, use the default data page size of 1MByte.

  • allow_truncated_timestamps (bool, default False) ‚Äì Allow loss of data when coercing timestamps to a particular resolution. E.g. if microsecond or nanosecond data is lost when coercing to ‚Äòms‚Äô, do not raise an exception. Passing allow_truncated_timestamp=True will NOT result in the truncation exception being ignored unless coerce_timestamps is not None.

  • flavor ({'spark'}, default None) ‚Äì Sanitize schema or set other compatibility options to work with various target systems.

  • use_byte_stream_split (bool or list, default False) ‚Äì Specify if the byte_stream_split encoding should be used in general or only for some columns.   • use_compliant_nested_type (bool, default False) ‚Äì

    Whether to write compliant Parquet nested type (lists) as defined here, defaults to False. For use_compliant_nested_type=True, this will write into a list with 3-level structure where the middle level, named list, is a repeated group with a single field named element:

    <list-repetition> group <name> (LIST) {
    repeated group list {
          <element-repetition> <element-type> element;
    }
}

    For use_compliant_nested_type=False, this will also write into a list with 3-level structure, where the name of the single field of the middle level list is taken from the element name for nested columns in Arrow, which defaults to item:

    <list-repetition> group <name> (LIST) {
    repeated group list {
        <element-repetition> <element-type> item;
    }
}
