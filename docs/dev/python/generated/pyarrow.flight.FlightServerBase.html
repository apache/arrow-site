A Flight service definition.

Override methods to define your Flight service.

Parameters

  • location (str, tuple or Location optional, default None) ‚Äì Location to serve on. Either a gRPC URI like grpc://localhost:port, a tuple of (host, port) pair, or a Location instance. If None is passed then the server will be started on localhost with a system provided random port.

  • auth_handler (ServerAuthHandler optional, default None) ‚Äì An authentication mechanism to use. May be None.

  • tls_certificates (list optional, default None) ‚Äì A list of (certificate, key) pairs.

  • verify_client (boolean optional, default False) ‚Äì If True, then enable mutual TLS: require the client to present a client certificate, and validate the certificate.

  • root_certificates (bytes optional, default None) ‚Äì If enabling mutual TLS, this specifies the PEM-encoded root certificate used to validate client certificates.

  • middleware (list optional, default None) ‚Äì A dictionary of ServerMiddlewareFactory items. The keys are used to retrieve the middleware instance during calls (see ServerCallContext.get_middleware()).
