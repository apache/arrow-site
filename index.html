What is Arrow?

Format

Apache Arrow defines a language-independent columnar memory format for flat and hierarchical data, organized for efficient analytic operations on modern hardware like CPUs and GPUs. The Arrow memory format also supports zero-copy reads for lightning-fast data access without serialization overhead.

Learn more about the format or read the specification.

Libraries

The Arrow project includes libraries that implement the memory specification in many languages. They enable you to use the Arrow format as an efficient means of sharing data across languages and processes. Libraries are available for C, C++, C#, Go, Java, JavaScript, MATLAB, Python, R, Ruby, and Rust.

See how to install and get started.

Applications

Arrow libraries provide a foundation for developers to build fast analytics applications. Many popular projects use Arrow to ship columnar data efficiently or as the basis for analytic engines.

The libraries also include built-in features for working with data directly, including Parquet file reading and querying large datasets. See more Arrow use cases.

Why Arrow?

Efficient

The Apache Arrow format allows computational routines and execution engines to maximize their efficiency when scanning and iterating large chunks of data. In particular, the contiguous columnar layout enables vectorization using the latest SIMD (Single Instruction, Multiple Data) operations included in modern processors.

Standard

Apache Arrow is backed by key developers of major open source projects, including Calcite, Cassandra, Drill, Hadoop, HBase, Ibis, Impala, Kudu, Pandas, Parquet, Phoenix, Spark, and Storm, making it the de-facto standard for columnar in-memory analytics.

Learn more about projects that are powered by Apache Arrow

Community

Apache Arrow is software created by and for the developer community. We are dedicated to open, kind communication and consensus decisionmaking. Our committers come from a range of organizations and backgrounds, and we welcome all to participate with us.
